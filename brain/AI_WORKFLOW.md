# AI Workflow

## Purpose
This file defines how AI collaborators should work in Anodizex.

## Workflow
- Start by reading `AGENTS.md`, `brain/BRAIN.md`, and task-relevant Brain files.
- Treat the copied `after-service` implementation as scaffolding, not final product truth.
- Prefer small, reviewable changes that progressively rename, delete, and adapt the codebase toward Anodizex.
- Update Brain docs whenever a durable decision or product behavior changes.

## Definition Of Done
- Relevant checks have been run or explicitly skipped with a reason.
- Brain documentation impact has been reviewed.
- Final response lists Brain files updated or explains why none were required.
