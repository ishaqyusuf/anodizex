# Sign-Out Cookie Variant Persistence

## Status
Fixed on 2026-06-10.

## Symptom
Clicking sign out could redirect to `/sign-in` but still leave the dashboard treated as authenticated.

## Root Cause
The UI manually posted to `/api/auth/sign-out`, bypassing Better Auth client-side session signaling. The server sign-out handler also only expired the session cookie variant generated by the current Better Auth `baseURL`/secure-cookie configuration, while the dashboard proxy intentionally accepts multiple historical and secure-prefixed session cookie names.

If a browser had a stale accepted variant such as `__Secure-better-auth.session_token`, sign-out could return success while the proxy still saw a session cookie.

## Fix
- Dashboard sign-out now uses `authClient.signOut()` so Better Auth can update local session state and broadcast sign-out.
- The auth route appends expiry cookies for all dashboard-recognized Better Auth session cookie variants after `/api/auth/sign-out`.
- The proxy reads accepted session cookie names from the same helper used by the sign-out fallback.
