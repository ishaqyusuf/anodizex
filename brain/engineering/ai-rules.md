# AI Rules

## Purpose
This file gives implementation rules for AI agents.

## Before Editing
- Read `brain/BRAIN.md`.
- Check `git status --short`.
- Inspect existing files before adding new ones.
- Identify the active roadmap phase.

## During Editing
- Keep work scoped to the requested phase.
- Use existing patterns before creating new abstractions.
- Update docs when changing architecture, schema, API, billing, auth, or roadmap.
- Preserve uncommitted user changes.

## After Editing
- Run relevant checks.
- Update task docs if status changed.
- Summarize what changed and what was verified.

## Forbidden Drift
- Do not rename remaining copied afterservice route/domain/cookie behavior outside a focused cleanup pass.
- Do not copy Plot Keys real estate concepts into product code.
- Do not wire real outbound messaging without explicit configuration.
- Do not make checkout redirects the source of paid entitlement truth.
